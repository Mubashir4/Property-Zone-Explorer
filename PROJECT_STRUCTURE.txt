KEY ARCHITECTURAL DECISIONS
===========================

1. API Layer Abstraction
   → All API calls go through service layer
   → Easy to switch between mock/real data
   → Centralized error handling

2. State Management Strategy
   → Redux Toolkit: UI state (filters, selections)
   → TanStack Query: Server state (API data, caching)
   → Clear separation of concerns

3. Component Organization
   → Feature-based structure (search, propertyDetail)
   → Reusable components in components/
   → Each component in its own folder

4. Type Safety
   → TypeScript strict mode
   → All props, state, and API responses typed
   → Zero 'any' types

5. Performance
   → Code splitting by route (React.lazy)
   → Marker clustering for 500+ properties
   → Debounced search inputs
   → React.memo for expensive components

6. Developer Experience
   → Hot Module Replacement (Vite)
   → Redux DevTools integration
   → React Query DevTools
   → ESLint + Prettier automation
